Codex workflow

我使用 Codex 修改项目的流程

  1. 确认 main 是最新状态，并且 GitHub Desktop 显示 No local changes。
  2. 创建一个新的试验分支，让 Codex 先阅读项目并给计划。
  3. 确认计划后再允许 Codex 修改文件。
  4. 在 GitHub Desktop 里检查 changed files 和 diff。
  5. 让 Codex review 这次改动，再决定 commit、discard 或继续修改。
  6. 通过 Pull Request 合并，并同步回本地 main。

Safety checklist

提交前我要特别检查

不要提交密钥 .env、API key、密码、个人隐私信息都不应该进入 GitHub。
不要盲目提交大改动 如果 Codex 改了很多文件，先 review，再决定下一步。
不要跳过 PR 检查 Files changed 是合并前最后一次看证据的地方。
